# Plan: Migrate from Poetry to uv

## Why uv?
- **10-100x faster** dependency resolution and installation vs Poetry
- **Drop-in replacement** for pip, pip-tools, and virtualenv — single binary
- **Native lockfile** (`uv.lock`) with cross-platform resolution
- **Built-in Python version management** (no need for pyenv)
- **PEP 621 compliant** — uses standard `[project]` table instead of Poetry-specific `[tool.poetry]`

## Current State
- Poetry with `pyproject.toml` (`[tool.poetry]` sections), `poetry.lock`, `poetry.toml`
- `requirements.txt` and `h100_requirements.txt` as pip fallbacks
- Dockerfile uses `pip install -r requirements.txt`
- No CI/CD workflows to update
- Tool configs (black, isort, pytest, mypy) in `pyproject.toml` — these stay as-is

---

## Migration Steps

### Step 1: Convert `pyproject.toml` to PEP 621 format
Replace Poetry-specific sections with standard metadata:

- `[tool.poetry]` → `[project]` with standard fields (`name`, `version`, `description`, `readme`, `license`, `requires-python`)
- `[tool.poetry.dependencies]` → `[project.dependencies]` (list format, not table)
- `[tool.poetry.group.dev.dependencies]` → `[dependency-groups]` dev group
- `[tool.poetry.scripts]` → `[project.scripts]`
- `[build-system]` → update to use `hatchling` (standard, uv-compatible backend)
- Add `[tool.uv]` section for uv-specific config (e.g., `package = true` for the `src/` layout)

### Step 2: Generate `uv.lock` and remove Poetry files
- Run `uv lock` to generate the new lockfile
- Delete `poetry.lock` and `poetry.toml`
- Verify resolution succeeds for all dependencies

### Step 3: Update `requirements.txt` files
- Replace `requirements.txt` with `uv export --no-dev --no-hashes -o requirements.txt` output (keeps Dockerfile compatibility)
- Keep `h100_requirements.txt` as-is (it's a standalone minimal list, not Poetry-managed)

### Step 4: Update Dockerfile
- Option A (minimal change): Keep `pip install -r requirements.txt` — works since we export from uv
- Option B (full uv): Install uv in Docker and use `uv sync --frozen --no-dev` for faster builds
- **Recommendation**: Option A for now (no Docker change needed), migrate Docker to uv later if desired

### Step 5: Update documentation
- `README.md`: Replace `poetry install` / `pip install` instructions with `uv sync`
- `tests/README.md`: Replace `poetry install --with dev` with `uv sync`
- `docs/CLOUD_GPU_SETUP.md`: Update install commands
- `H100_SETUP.md`: No changes needed (uses its own minimal requirements)

### Step 6: Verify
- Run `uv sync` to create venv and install all deps
- Run `uv run pytest` to confirm tests pass
- Confirm `uv run python main.py --help` works
- Confirm scripts (`archive_transcriber`, `vtt_to_ttml`) resolve correctly

## Risks & Notes
- `torch` is a large dependency — uv handles it well but initial `uv lock` may take a moment
- The `packages = [{include = "python", from = "src"}]` Poetry layout maps to a standard `[tool.hatch.build.targets.wheel]` sources config
- All `[tool.*]` configs (black, isort, pytest, mypy) are standard and remain unchanged
- `h100_requirements.txt` is independent and unaffected
