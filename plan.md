# Plan: Migrate from Poetry to uv + Consolidate Dev Tools with Ruff

## Why uv?
- **10-100x faster** dependency resolution and installation vs Poetry
- **Drop-in replacement** for pip, pip-tools, and virtualenv â€” single binary
- **Native lockfile** (`uv.lock`) with cross-platform resolution
- **Built-in Python version management** (no need for pyenv)
- **PEP 621 compliant** â€” uses standard `[project]` table instead of Poetry-specific `[tool.poetry]`

## Why consolidate to ruff?
- ruff is already a dev dependency but **unconfigured and unused**
- ruff replaces **black** (formatter), **isort** (import sorting), and **flake8** (linter) â€” all three currently installed
- Single tool, written in Rust, orders of magnitude faster
- Drops 3 dev dependencies and their transitive deps

## Current State
- Poetry with `pyproject.toml` (`[tool.poetry]` sections), `poetry.lock`, `poetry.toml`
- `requirements.txt` and `h100_requirements.txt` as pip fallbacks
- Dockerfile uses `pip install -r requirements.txt`
- No CI/CD workflows to update
- Dev tools: black (configured), isort (configured), flake8 (installed, no config), ruff (installed, no config), mypy (configured)

## Migration Steps

### Step 1: Convert `pyproject.toml` to PEP 621 format
Replace Poetry-specific sections with standard metadata:

- `[tool.poetry]` â†’ `[project]` with standard fields (`name`, `version`, `description`, `readme`, `license`, `requires-python`)
- `[tool.poetry.dependencies]` â†’ `[project.dependencies]` (list format, not table)
- `[tool.poetry.group.dev.dependencies]` â†’ `[dependency-groups]` dev group
- `[tool.poetry.scripts]` â†’ `[project.scripts]`
- `[build-system]` â†’ update to use `hatchling` (standard, uv-compatible backend)
- Add `[tool.uv]` section for uv-specific config (e.g., `package = true` for the `src/` layout)

### Step 2: Consolidate dev tools â€” replace black + isort + flake8 with ruff
- Remove `black`, `isort`, `flake8` from dev dependencies
- Replace `[tool.black]` and `[tool.isort]` configs with `[tool.ruff]`:
  - `line-length = 120`
  - `target-version = "py310"`
  - Enable `ruff format` (replaces black)
  - Enable isort rules (`I`) and flake8 rules (`E`, `F`, `W`) in `[tool.ruff.lint]`
  - Set `[tool.ruff.lint.isort]` profile to match current isort config
- Keep `mypy` and `pytest` as-is (ruff doesn't replace type checking or testing)

### Step 3: Generate `uv.lock` and remove Poetry files
- Run `uv lock` to generate the new lockfile
- Delete `poetry.lock` and `poetry.toml`
- Verify resolution succeeds for all dependencies

### Step 4: Update `requirements.txt` files
- Replace `requirements.txt` with `uv export --no-dev --no-hashes -o requirements.txt` output (keeps Dockerfile compatibility)
- Keep `h100_requirements.txt` as-is (it's a standalone minimal list, not Poetry-managed)

### Step 5: Update Dockerfile
- Option A (minimal change): Keep `pip install -r requirements.txt` â€” works since we export from uv
- Option B (full uv): Install uv in Docker and use `uv sync --frozen --no-dev` for faster builds
- **Recommendation**: Option A for now (no Docker change needed), migrate Docker to uv later if desired

### Step 6: Update documentation
- `README.md`: Replace `poetry install` / `pip install` instructions with `uv sync`
- `tests/README.md`: Replace `poetry install --with dev` with `uv sync`
- `docs/CLOUD_GPU_SETUP.md`: Update install commands
- `H100_SETUP.md`: No changes needed (uses its own minimal requirements)

### Step 7: Verify
- Run `uv sync` to create venv and install all deps
- Run `uv run ruff check .` and `uv run ruff format --check .` to confirm ruff works
- Run `uv run pytest` to confirm tests pass
- Confirm `uv run python main.py --help` works
- Confirm scripts (`archive_transcriber`, `vtt_to_ttml`) resolve correctly

## Risks & Notes
- `torch` is a large dependency â€” uv handles it well but initial `uv lock` may take a moment
- The `packages = [{include = "python", from = "src"}]` Poetry layout maps to a standard `[tool.hatch.build.targets.wheel]` sources config
- ruff format is nearly 100% compatible with black â€” minor whitespace diffs possible but unlikely with `line-length = 120`
- `h100_requirements.txt` is independent and unaffected
